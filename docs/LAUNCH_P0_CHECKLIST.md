# P0 上线检查清单

## 1. 内容安全

已接入云函数内容安全审核：

- 鱼塘动态：正文、标题、心情、标签、图片
- 鱼塘评论：评论内容
- 下班约局：标题、时间、地点、说明
- 约局评论：评论内容
- 福利申诉：申诉内容
- 用户资料：昵称、头像文字

云函数返回这些错误时，小程序端不得本地兜底发布：

- `CONTENT_RISK`：内容或图片不适合发布
- `REVIEW_FAILED`：内容安全服务暂时不可用
- `BAD_REQUEST`：参数不完整
- `FORBIDDEN`：无操作权限

## 2. 云函数部署

每次修改内容安全逻辑后，需要重新部署云函数：

1. 右键 `cloudfunctions/api`
2. 选择“上传并部署：云端安装依赖”
3. 确认 `cloudfunctions/api/config.json` 一起上传
4. 在开发者工具里测试发布动态、评论、约局、约局评论

## 3. 隐私保护指引

在微信公众平台配置小程序用户隐私保护指引，至少覆盖：

- 用户昵称：用于动态、评论、约局展示
- 用户头像或头像文字：用于动态、评论、个人页展示
- 图片或视频：用于鱼塘动态图片上传和展示
- 位置信息：用于下班约局选择集合地点和打开微信地图导航
- 用户发布内容：用于鱼塘动态、评论、下班约局、申诉反馈
- 云存储：用于保存动态图片和用户发布内容

不要在隐私说明里承诺“升级提现”或“真钱奖励”。第一版推荐描述为：

- 积分用于宠物成长、身份展示、福利活动参与资格
- 福利活动以具体活动规则为准，不承诺固定现金收益

## 4. 数据库集合

上线前确认云开发控制台已有：

- `users`
- `fish_posts`
- `meetups`
- `point_events`
- `welfare_appeals`

建议索引：

- `fish_posts`：`deleted + reported + createdAt`
- `fish_posts`：`_openid + deleted + reported + createdAt`
- `fish_posts`：`channel + deleted + reported + createdAt`
- `meetups`：`deleted + createdAt`
- `meetups`：`type + deleted + createdAt`
- `point_events`：`_openid + date + eventType`

## 5. 提审前回归

必须手动走通：

- 发布一条正常鱼塘动态
- 上传一张正常图片
- 评论鱼塘动态
- 删除自己的动态
- 举报一条动态
- 发布一个下班约局
- 报名约局
- 取消报名
- 评论约局
- 提交福利申诉
- 用疑似违规文本测试拦截提示

## 6. 上线策略

第一版不要上线现金提现、现金承诺、抽奖式奖励。可以保留：

- 宠物等级
- 积分成长
- 福利池入口
- 奶茶券、运动券、会员券等活动资格

等社区内容稳定、审核流程跑通后，再设计更完整的福利规则。
