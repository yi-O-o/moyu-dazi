# 摸鱼搭子功能开发计划

## 1. 当前目标

把小程序从单纯的工资计时工具，升级成一个“上班陪伴 + 下班期待 + 宠物成长”的轻游戏产品。

本阶段优先完成这些功能：

- 首页 3D 宠物陪伴
- 下班许愿池
- 玄学一下
- 积分与宠物等级
- 搭子圈行为加分
- 宠物福利池规则预埋

暂不做上班任务管理，避免产品变成待办工具。

## 2. 功能范围

### 2.1 首页

首页保留这些核心信息：

- 今日已赚
- 离下班时间
- 宠物成长积分
- 下班许愿池
- 玄学一下
- 快捷入口：宠物小窝、摸鱼一下、玄学一下、下班找搭子

首页不放：

- 今日工作任务
- 打工人格测试
- 复杂表单

### 2.2 下班许愿池

用户可以添加今天下班想做的事情。

基础能力：

- 输入愿望，最多 24 个字
- 添加愿望
- 展示最近 12 个愿望
- 删除愿望
- 首次添加愿望获得宠物经验 +1

后续能力：

- 根据愿望推荐搭子活动
- 根据愿望推荐吃饭、运动、娱乐内容
- 宠物根据愿望生成陪伴文案

### 2.3 玄学一下

用户点击后抽一支“今日工位签”。

产品口径：

- 今日工位签不作为鱼塘帖子发布。
- 今日工位签像微信状态一样展示在个人主页头像/昵称旁边。
- 别人从鱼塘头像进入个人主页后，可以看到该用户的今日工位签。
- 今日工位签一天只能抽取一次，抽取后不可换签。

签文包含：

- 今日状态，例如小吉、中吉、大吉、平安
- 今日宜做
- 今日描述
- 幸运动作
- 今日忌做

积分规则：

- 每天首次抽签 +1 分
- 当天重复抽签可以看内容，但不重复加分

### 2.4 摸鱼一下

摸鱼不是任务，而是合理休息。

规则：

- 每天首次点击 +2 分
- 重复点击只提示今天已经记过
- 后续可加入 3 分钟倒计时、呼吸动画、随机摸鱼文案

### 2.5 积分与宠物等级

积分统一作为宠物经验。

当前积分来源：

- 添加下班愿望：+1，每天 1 次
- 玄学一下：+1，每天 1 次
- 摸鱼一下：+2，每天 1 次
- 搭子圈发布：+3，每天最多 3 次
- 搭子圈评论：+1，每天最多 5 次
- 报名活动：+2，每天最多 3 次
- 分享下班计划：+2，每天最多 2 次

等级配置：

- Lv.1 试用期小搭子：0 分
- Lv.2 准点喝水员：5 分
- Lv.3 工位陪伴官：12 分
- Lv.4 摸鱼观察员：22 分
- Lv.5 下班愿望守护者：35 分
- Lv.6 搭子圈活跃星：55 分
- Lv.7 打工生存专家：80 分
- Lv.8 宠物福利合伙人：110 分

### 2.6 搭子圈加分

下班页先用演示按钮模拟搭子圈行为。

后续真实搭子圈上线后，需要把这些行为接入同一套积分系统：

- 发布搭子
- 评论
- 报名活动
- 分享下班计划

需要防刷：

- 每日上限
- 同内容重复检测
- 异常频率限制
- 被删除内容不计入长期贡献

### 2.7 真钱福利池

不做“升级直接提现”。

推荐做成福利池：

- 等级达到门槛后获得参与资格
- 奖励可以是奶茶券、运动券、会员券、小额红包
- 现金奖励必须有明确活动规则、预算池、次数上限和风控策略
- 文案使用“有机会获得”，避免承诺必得

上线前需要补充：

- 活动规则页
- 隐私协议
- 反作弊规则
- 平台审核说明

### 2.8 上班摸鱼动态

鱼塘的核心不是严肃职场交流，而是“展示我的上班生活”和“看看别人怎么熬班”。

第一批鱼塘频道：

- 杯子鱼塘：晒上班喝的杯子、水杯、咖啡杯、保温杯、奶茶。
- 工位鱼塘：晒工位、桌搭、键盘、显示器、收纳、桌面小物。
- 上班穿搭：晒通勤穿搭、工牌穿搭、办公室舒适穿搭。
- 午饭鱼塘：晒工作日午饭、外卖、便当、公司食堂。
- 通勤鱼塘：晒地铁、公交、骑车、停车、通勤路上的小事。
- 摸鱼角落：晒摸鱼方式、工位小零食、短暂休息。
- 宠物搭子：晒宠物等级、装扮、气泡文案和今日互动。
- 下班愿望：把许愿池愿望发出来，找同城或同兴趣的人响应。

每个频道都要有明确的内容模板，降低发布成本。

发布模板：

- 图片
- 一句话正文
- 频道

互动能力：

- 点赞
- 评论
- 收藏
- 同款/想要
- 问链接
- 报名一起做
- 举报

鱼塘积分：

- 发布展示内容：+2，每天最多 3 次。
- 评论：+1，每天最多 5 次。
- 被点赞：每 5 个赞 +1，每天最多 3 分。
- 内容被收藏：+1，每天最多 3 分。
- 优质内容入选精选：+5。

内容安全：

- 图片和文本需要审核。
- 不允许出现公司敏感信息、工牌敏感信息、客户资料、屏幕业务内容。
- 工位图需要提醒用户遮挡屏幕、姓名、工牌、文件。
- 鱼塘要有举报和删除能力。

### 2.9 鱼塘频道优先级

第一阶段先做最容易出内容的频道：

1. 杯子鱼塘
2. 工位鱼塘
3. 上班穿搭

第二阶段扩展：

4. 午饭鱼塘
5. 通勤鱼塘
6. 摸鱼角落

第三阶段联动宠物和搭子：

7. 宠物搭子
8. 下班愿望

原因：

- 杯子、工位、穿搭最适合图片展示。
- 发布成本低，用户不用写长文。
- 容易形成“同款”“求链接”“好看”“哪个公司这么爽”的讨论。
- 能自然接入积分和宠物成长。

### 2.10 鱼塘 MVP 页面

建议新增一级页面或 Tab：

- 鱼塘首页
- 频道详情
- 发布内容
- 帖子详情
- 我的鱼塘

鱼塘首页：

- 顶部频道：杯子、工位、穿搭、午饭、通勤、摸鱼、宠物、愿望
- 精选内容流
- 最新发布
- 发帖按钮

频道详情：

- 频道介绍
- 发布模板提示
- 内容列表
- 热门标签

发布内容：

- 选择频道
- 上传图片
- 一句话正文
- 发布前安全提醒

帖子详情：

- 图片展示
- 正文
- 点赞、收藏、评论
- 同款/求链接按钮
- 举报

我的鱼塘：

- 我的发布
- 我的评论
- 我的收藏
- 我的积分贡献

### 2.11 频道发布模板

杯子鱼塘：

- 标题提示：今天靠什么续命？
- 正文提示：这只杯子/这杯饮料有什么故事？
- 标签建议：咖啡、奶茶、保温杯、续命、同款求链接

工位鱼塘：

- 标题提示：晒晒今天的工位角落
- 正文提示：桌面上最治愈你的东西是什么？
- 标签建议：键盘、显示器、桌搭、收纳、工位小物
- 安全提醒：发布前请遮挡屏幕、工牌、文件和公司敏感信息。

上班穿搭：

- 标题提示：今日通勤穿搭
- 正文提示：今天这套适合开会、通勤还是摸鱼？
- 标签建议：通勤、舒适、开会、工牌、显瘦

午饭鱼塘：

- 标题提示：今天中午吃什么
- 正文提示：好吃吗，值不值，吃完困不困？
- 标签建议：外卖、便当、食堂、减脂、碳水快乐

通勤鱼塘：

- 标题提示：今天通勤路上发生了什么
- 正文提示：堵车、地铁、骑车、路边风景都可以。
- 标签建议：地铁、公交、骑车、堵车、早高峰

摸鱼角落：

- 标题提示：你今天怎么回血？
- 正文提示：小零食、散步、喝水、发呆都算。
- 标签建议：零食、休息、带薪呼吸、办公室生存

宠物搭子：

- 标题提示：晒晒我的宠物搭子
- 正文提示：今天宠物说了什么，升到几级了？
- 标签建议：宠物等级、装扮、气泡、陪伴

下班愿望：

- 标题提示：今天下班想做什么
- 正文提示：有人一起吗？
- 标签建议：约饭、运动、电影、散步、躺平

## 4. 开发阶段

### 阶段一：功能收口

目标：把当前功能稳定下来。

任务：

- 确认首页不再展示今日任务和人格测试
- 确认许愿池可添加、展示、删除
- 确认玄学一下可抽签
- 确认摸鱼一下可加分
- 确认宠物等级在首页和宠物页一致
- 确认下班页搭子行为能加分

验收：

- 所有 JS 语法检查通过
- 微信开发者工具无运行报错
- 删除愿望后页面立刻刷新
- 同一天重复加分会被限制

### 阶段二：体验优化

目标：让功能更像产品，而不是演示。

任务：

- 玄学签结果当天固定，刷新后不变
- 愿望池空状态优化
- 愿望删除增加二次确认
- 宠物根据用户行为说不同的话
- 积分增加时添加轻量动画反馈
- 首页布局检查，避免宠物遮挡文字

验收：

- 当天抽签多次结果一致
- 删除愿望不会误触
- 宠物文案能响应愿望、玄学、摸鱼

### 阶段三：摸鱼广场 MVP

目标：先做出能发、能看、能互动的上班摸鱼动态。

任务：

- 新增鱼塘入口
- 新增频道页：杯子、工位、穿搭
- 新增发布表单
- 支持图片、标题、正文、标签
- 支持点赞、评论、收藏
- 发布、评论接入积分
- 加入基础举报入口

验收：

- 用户能发布一条杯子/工位/穿搭内容
- 用户能浏览不同频道
- 用户能点赞和评论
- 发布和评论能加分
- 敏感信息提醒文案展示明确

### 阶段四：下班约局真实化

目标：把下班页从演示按钮升级为真实搭子动态雏形。

任务：

- 约局发布表单
- 约局列表
- 评论列表
- 报名按钮
- 发布/评论/报名接入积分
- 加入每日上限和重复内容校验

验收：

- 用户能发布一条搭子需求
- 用户能评论
- 用户能报名
- 积分规则和上限正常

### 阶段五：鱼塘增强

目标：让鱼塘从展示流升级成讨论和发现。

任务：

- 增加频道筛选
- 增加热门标签
- 增加同款/求链接按钮
- 增加精选内容
- 增加个人主页
- 鱼塘头像进入个人主页
- 个人主页展示今日工位签
- 增加我的发布、我的评论、我的收藏
- 增加图片安全提示

验收：

- 用户能按频道和标签浏览
- 用户能查看别人主页
- 用户能沉淀自己的发布和收藏
- 运营能标记精选内容

### 阶段六：宠物成长增强

目标：让积分真正有吸引力。

任务：

- 宠物等级解锁不同气泡文案
- 宠物等级解锁装扮
- 宠物页展示升级奖励
- 首页展示距离下一等级还差多少分
- 增加成就徽章

验收：

- 升级时有明确反馈
- 不同等级有不同可见权益
- 用户知道为什么要继续攒分

### 阶段七：福利池预研

目标：为真钱/券类奖励做合规准备。

任务：

- 设计福利池规则
- 设计活动规则页
- 设计预算和发放上限
- 设计反作弊策略
- 设计领奖流程

验收：

- 福利规则不承诺必得现金
- 每项奖励有明确预算和发放条件
- 有异常用户拦截策略

## 6. 风险与注意事项

- 不要让首页变成任务管理工具。
- 现金奖励不要写成“升级提现”。
- 搭子圈积分必须有每日上限。
- 鱼塘积分必须有每日上限。
- 评论、发布等鱼塘行为需要反刷机制。
- 工位、杯子、穿搭等图片内容需要提醒用户遮挡敏感信息。
- 鱼塘需要举报、删除和审核流程。
- 宠物和浮层不能遮挡核心工资信息。
- 所有奖励文案要保持轻松，但不能误导用户。

## 7. 第一版上线前收口清单

### 7.1 必须先改

1. 鱼塘首页内容流优先

- 当前鱼塘首页先展示 hero、频道小馆、摸鱼热榜，动态列表位置偏后。
- 第一版应参考朋友圈、小红书、豆瓣小组等内容产品，用户进入后先看到动态流。
- 调整建议：顶部只保留轻标题、当前等级和发动态入口；动态流前置；频道和热榜下移或折叠。

2. 云开发后端正式接入页面

- 当前已经有 `cloudfunctions/api` 和 `utils/cloudApi.js`，但页面仍主要使用本地缓存。
- 上线前必须让不同用户能看到真实动态、约局、评论和报名。
- 接入顺序：鱼塘列表/发布/评论/删除；下班约局列表/发布/报名/取消；我的页数据；积分等级。

3. 下班约局发布表单轻量化

- 当前用户要填标题、时间、人数、地点、说明，发布成本偏高。
- 调整建议：增加快捷模板，例如约饭、打球、散步、电影；点击模板自动填类型和标题，用户只补时间和地点。

4. 个人页信息收口

- 当前个人页同时展示宠物、福利池、我的约局、鱼塘动态、徽章、第二版功能预告。
- 第一版建议隐藏“第二版 AI 创造歌曲”等未来能力，避免用户误以为当前可用。

### 7.2 体验优化

1. 鱼塘发布入口固定化

- 当前发动态按钮在 hero 区域，用户下滑后看不到。
- 调整建议：右下角固定一个 `+` 悬浮发布按钮，贴近主流内容社区操作习惯。

2. 删除和举报入口统一

- 当前自己的动态显示删除，别人动态通过 `···` 举报。
- 调整建议：详情页点击 `···` 后弹 action sheet，选项包括删除、举报、取消；列表页也保持一致。

3. 首页主动作减少

- 当前首页包含工资、宠物、许愿池、玄学、摸鱼、鱼塘、约局等多个入口。
- 第一版建议首页主动作聚焦：摸鱼一下、发动态、下班找搭子。
- 玄学和许愿池可以保留，但视觉层级下调。

4. 名称统一

- 当前存在鱼塘、摸鱼广场、上班动态池等多个叫法。
- 第一版统一使用“鱼塘”，页面标题可用“上班人的鱼塘”。

### 7.3 上线前必备

1. 微信头像昵称授权或默认头像昵称策略。
2. 图片上传接入云存储，不能依赖本地临时路径。
3. 文本和图片内容安全审核。
4. 页面加载态、失败态、空状态补齐。
5. 云数据库权限规则配置。
6. 帖子详情和约局详情支持分享给微信好友。

### 7.4 推荐执行顺序

1. 个人主页资料编辑。
2. 鱼塘首页改成内容流优先。
3. 鱼塘发动态改成固定悬浮 `+`。
4. 下班约局发布增加快捷模板。
5. 图片上传接入云存储后的内容安全审核。
6. 云数据库权限规则细化。
7. 福利池后台处理进度。
8. 下班约局到场确认。
